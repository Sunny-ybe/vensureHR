# Bench — Talent Intelligence for VensureHR

> *Every interview is an asset. We make sure it compounds.*

---

## The Problem

Recruiters spend hours sourcing, screening, and interviewing candidates. They build real opinions — this person is sharp, this one just needs six more months of experience. Then the role gets filled, the requisition closes, and all of that knowledge gets buried in an ATS that nobody opens again.

Six months later a new role opens. The recruiter starts from scratch. Posts the job. Sources again. Screens again. As if those conversations never happened.

Meanwhile that candidate spent the last six months shipping projects, getting promoted, learning exactly what the company needed. The recruiter has no idea. And neither does the candidate.

**That's what Bench fixes.**

---

## What We Built

Bench is a living talent intelligence platform that watches your past candidates grow over time — their GitHub commits, LinkedIn posts, career moves — and surfaces them automatically when the right role opens.

Built specifically for VensureHR's three service lines: **PEO**, **ASO**, and **HCM**.

---

## Features

### Talent Pool
- Dashboard of every past candidate with real-time match scores
- Growth trajectory — see how a candidate's score has changed since their interview
- Natural language search: *"Who has been building AI projects in the last 6 months?"*

### Candidate Intelligence
- GitHub activity — real repos and commit history pulled via GitHub API
- LinkedIn posts — candidate-consented professional activity
- Twitter/X — public posts and engagement
- Interview transcript — stored and queryable
- AI Summary — GPT-4o generated summary of everything we know

### Chat with Any Candidate
Ask anything about a specific candidate in natural language:
- *"What has this candidate shipped since we interviewed them?"*
- *"Do they have experience with distributed systems?"*
- *"What would they need to learn to qualify for this role?"*

Powered by Supermemory — one memory space per candidate containing their entire professional history.

### Smart Matching
- Cosine similarity matching between job descriptions and candidate profiles using OpenAI embeddings
- GPT-4o generated reasoning for every match score
- Configurable scoring weights — tune Technical Skills, Growth Trajectory, and Culture Fit to your priorities
- Scores update automatically as candidates grow

### Re-engagement
- One-click personalized outreach email generated by GPT-4o
- Email references the candidate's recent activity and the specific role
- Includes upskilling recommendations: *"To qualify for this role, consider learning X and Y"*
- Sent directly via Resend API to the candidate's email on file

### Service Line Matching
Candidates automatically categorized across VensureHR's service lines:
- **PEO** — Professional Employer Organization
- **ASO** — Administrative Services Organization  
- **HCM** — Human Capital Management

## Tech Stack

### Backend
- **FastAPI** — async Python API, 10 endpoints
- **PostgreSQL** — primary database on Supabase
- **SQLAlchemy** — async ORM with asyncpg driver
- **Supermemory** — one memory space per candidate for NL querying
- **OpenAI** — embeddings (text-embedding-3-small) + GPT-4o for matching, summaries, emails
- **GitHub REST API** — real repo and commit data
- **Resend** — transactional email for re-engagement
- **Celery + Redis** — background scrape jobs on 14-day schedule
- **python-dotenv** — environment management

### Frontend
- **Next.js 14** — App Router
- **Tailwind CSS** — styling
- **Vercel** — deployment

```
│   VensureHR     │     │   Candidate     │

│  (Next.js)      │     │  (Next.js)      │
└────────┬────────┘     └────────┬────────┘
         │                       │
         └──────────┬────────────┘
                    │
         ┌──────────▼────────────┐
         │     FastAPI Backend    │
         │   (10 REST endpoints)  │
         └──┬──────┬──────┬──────┘
            │      │      │
     ┌──────▼─┐ ┌──▼───┐ ┌▼──────────┐
     │Postgres│ │Supra │ │  OpenAI   │
     │Supabase│ │memory│ │  GPT-4o   │
     └────────┘ └──────┘ └───────────┘
```

### Data Flow
1. Candidate signs up → provides LinkedIn URL, GitHub handle, Twitter, consents
2. FastAPI creates candidate record in PostgreSQL + Supermemory space
3. Background job scrapes GitHub every 14 days → feeds into Supermemory
4. Candidate can manually feed LinkedIn posts and tweets via candidate portal
5. VensureHR queries candidates in natural language → Supermemory returns relevant memories → GPT-4o synthesizes answer

## Database Schema

```sql
companies         — VensureHR and client companies
candidates        — past interview candidates with consent tracking
jobs              — open positions per company
matches           — computed match scores (upserted on recompute)
upskill_recs      — per-candidate, per-job skill gap recommendations
scrape_logs       — every scrape attempt with SHA256 dedup hash
candidate_snapshots — raw scraped content as JSONB
```

**Key design decisions:**
- `consent_at` is immutable — written once, never modified (legal audit trail)
- `UNIQUE(job_id, candidate_id)` on matches — upsert on recompute, no duplicates
- PostgreSQL owns structured truth, Supermemory owns temporal unstructured activity
- SHA256 content hashing prevents duplicate memories on re-scrape

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/candidates` | List all candidates with match scores |
| GET | `/candidates/:id` | Full candidate profile with Supermemory data |
| POST | `/candidates/:id/chat` | Ask anything about a candidate |
| POST | `/candidates/:id/sync-github` | Pull real GitHub data into Supermemory |
| POST | `/candidates/:id/feed` | Feed LinkedIn post or tweet into Supermemory |
| POST | `/candidates/:id/transcript` | Store interview transcript |
| POST | `/candidates/:id/reengage` | Generate and send re-engagement email |
| POST | `/query` | NL search across all candidates |
| POST | `/matches` | Compute match score for job + candidate |
| GET | `/jobs` | List open positions |

## Setup

### Prerequisites
- Python 3.12+
- Node.js 18+
- Supabase account
- OpenAI API key
- Supermemory API key
- GitHub personal access token
- Resend API key

### Backend

```bash
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

Create `backend/.env`:
```
DATABASE_URL=postgresql+asyncpg://...
OPENAI_API_KEY=sk-...
SUPERMEMORY_API_KEY=sm_...
GITHUB_TOKEN=ghp_...
RESEND_API_KEY=re_...
```

Run schema:
```bash
# Paste schema.sql into Supabase SQL Editor and run
```

Seed database:
```bash
python3 seed.py
python3 seed_supermemory.py
```

Start backend:
```bash
python3 -m uvicorn main:app --reload --port 8000
```

### Frontend

```bash
cd frontend
npm install
```

Create `frontend/.env.local`:
```
NEXT_PUBLIC_API_URL=http://localhost:8000
```

Start frontend:
```bash
npm run dev
```
